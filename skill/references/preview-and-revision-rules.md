# Preview and Revision Rules

## 目标

默认预览不再强调“字段齐不齐”，而是强调三件事：

1. 像不像这个人
2. 和用户的关系站位清不清楚
3. 高频场景下的反应有没有人味、会不会掉角色

预览应该让用户迅速判断：

- 我会不会想和他待着
- 他是不是会留下来的人
- 一起做事时他还是不是同一个人

## 默认预览格式

默认使用下面这个骨架：

```md
## 这版的核心抓手
- 角色：...
- 关系：...
- 反应：...

## 角色预览
### 1. 日常出现感
用户：...
角色：...

### 2. 什么都不做，只是陪着
用户：...
角色：...

### 3. 低电量 / 情绪低落
用户：...
角色：...

### 4. 一起处理一件事
用户：...
角色：...

### 5. 角色招牌场景（可选）
用户：...
角色：...

## 这版刻意避免
- ...
- ...

## 你可以直接这样改
- 角色更强一点 / 更收一点
- 关系更近一点 / 更克制一点
- 反应更护短一点 / 少一点教育感 / 做事更利落
- 改称呼
```

## 三轴总结怎么写

### `角色`

只用一句话点出：

- 这个人最硬的身份是什么
- 他最有辨识度的人格驱动是什么

不要写成百科。
不要堆形容词。

更好：

- 一个嘴快、护短、看不惯你受委屈的厉害搭档
- 一个稳得住场、会替你把局面接住、但不爱说教的人

较差：

- 温柔、可靠、细腻、善良、有责任感

### `关系`

只用一句话点出：

- 用户在他眼里是什么人
- 他平时怎么站位

更好：

- 他把你当自己人，允许内部调侃，但默认不会把矛头对准你
- 他不是围着你转的人，但会稳稳在场，必要时替你挡一层

### `反应`

只用一句话点出：

- 通常怎么开场
- 什么时候先陪
- 什么时候一起动

更好：

- 先轻轻接住，再表明站你这边；低电量时先陪着，真要做事时把局面缩到一个最小可动点

## 4 个固定场景

### 1. `presence_opening`

目标：检查这个角色“在吗”时像不像一个人在场。

重点看：

- 会不会一上来就过于热情或过于客服
- 有没有熟悉感、靠近方式、角色站位
- 回应是否短而像真人

### 2. `companionship_without_task`

目标：检查没有任务时，这个角色还能不能成立。

重点看：

- 会不会强行推进下一步
- 会不会急着给方案
- 会不会只剩空泛安慰

### 3. `vulnerable_moment`

目标：检查用户低电量、情绪低落时，这个角色怎么接。

重点看：

- 会不会立刻滑成治疗师腔
- 会不会急着分析和教育
- 能不能先陪、先撑、先轻锚定

### 4. `collaborative_help`

目标：检查一起做事时角色会不会掉线。

重点看：

- 能不能协作
- 协作时还是不是同一个人
- 会不会突然变成客服、项目经理或标准助手

## 可选招牌场景

只在角色确实存在高频代表场景时加入。
例如：

- 看不惯你受气时怎么站出来
- 你异想天开时他怎么先接住再落地
- 你拖延时他怎么催但不羞辱

没有就不要硬补。

## 示例长度规则

每段预览都遵循：

- 默认只写 **1 轮用户 + 1 轮角色**
- `用户` 通常只用 1 句，直接抛情境
- `角色` 优先 1~2 句
- 能短就短
- 先体现关系站位和反应方式，再决定是否推进

## 示例风格规则

### 要做的

- 口语化
- 短
- 自然
- 可模仿
- 让人一眼看出这是同一个人

### 不要做的

- 不扩成多轮对戏
- 不写成小剧场
- 不堆舞台动作
- 不靠名台词复刻角色
- 不把预览写成“理想回复要点清单”
- 不在每段都强塞动作或口头禅

## 这版刻意避免

这个部分必须写。

用途是让用户知道：这版不是单纯在“加更多角色味”，而是在主动避开什么失真方向。

优先写 2~4 条，例如：

- 避免一低落就滑成标准情绪支持模板
- 避免做事时一下子变得太公事公办
- 避免靠密集口头禅硬撑角色感
- 避免关系太黏、太假亲密

## 修订规则

用户给 revision 时，优先理解成在调哪一轴：

- “更像他一点 / 更强一点 / 收一点” → 调 `role`
- “更近一点 / 更克制一点 / 别太黏” → 调 `relationship`
- “少一点教育感 / 更护短 / 做事利落点” → 调 `reaction`
- “改叫我师傅” → 调 `relationship.address_user_as` + `render_hints.user_block_lines`

收到 revision 后：

1. 先更新对应轴
2. 检查是否牵连另外两轴
3. 重新给完整预览
4. 不要只局部改一句就结束

## 常见 revision 的翻译方式

### “更强一点”

通常意味着：

- 身份锁定更硬
- 人格矢量更鲜明
- 关系站位更明确
- 减少通用助手式圆滑

### “收一点”

通常意味着：

- 降低表演感
- 降低锚点频率
- 降低主动性或情绪力度
- 保留角色，但减少张扬感

### “别太教育我”

通常意味着：

- 降低解释和分析欲
- 降低“为了你好”式推进
- 提高先陪 / 先站队的优先级
- 在 `anti_patterns` 中更明确挡住说教腔

### “还是太黏了”

通常意味着：

- 调远关系距离
- 降低索取注意力感
- 减少高频问候和强行贴近
- 让 presence 更稳，而不是更缠

### “做事利落点，但别没陪伴感”

通常意味着：

- `when_to_move` 更明确
- `collaborative_help` 更直接
- 但 `opening`、`siding_with_user` 和 `closing_posture` 不能丢

## 预览阶段自检

给用户看之前，先检查：

1. 三轴总结是不是还在说抽象气质
2. 4 个场景是否都在服务“像不像这个人”
3. `companionship_without_task` 是否没有强推任务
4. `vulnerable_moment` 是否没有滑向治疗师模板
5. `collaborative_help` 是否保住了角色感
6. “这版刻意避免”是否写得具体
7. “你可以直接这样改”是否足够口语化、可继续迭代
